# The Usual PASM specification

This directory is the source of truth for The Usual's system boundaries and
gameplay design.

- `core/foundation.yaml` — project systems and MVP contracts.
- `core/decisions.yaml` — accepted choices from the initial design session.
- `mvp/cottage-arrival.yaml` — the first headless vertical slice.
- `milestones/` — proposed implementation horizons after the MVP.
- `roadmap/` — long-term direction and explicit deferrals.
- `setting/setting-guide.yaml` — setting and tone constraints.

Run `uv run pasm validate` after every model change.
